Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

First off, I would advise my potential client to enable HTTPS instead of just HTTP, as HTTPS will encrypt the communication between the server and the client by using either TLS or SSL. I would also advise them to have a password policy, where they encourage strong passwords and use a two-factor authenticator.

I would encourage the use of a CI/CD tool which uses an automatic process which runs security checks to changes made to codes before they enter the codebase. This will help detect any vulnerabilities early in the process.

Sanitization of data is also something I would advise them to do. This means that the data coming in to the system, either from users, APIs or other web services is modified or filtered to prevent malicious code to enter the system. You can do this manually, but there are many third-party solutions you can use. When using a third-party provider, you should always use those who are well known and trustworthy.

At last I would encourage them to keep up with the OWASP list, which gives an overview if the most “popular” web security threats and offer guidance for improvements of vulnerabilities.
